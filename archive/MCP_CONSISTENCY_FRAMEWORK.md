# MCP Consistency Framework
**Purpose:** Ensure consistent, standardized, precise work without drift or assumptions

---

## Core Principles

### 1. **No Assumptions Policy**
- ❌ NEVER guess or assume requirements
- ✅ ALWAYS ask for clarification when unclear
- ✅ Document all decisions and rationale
- ✅ Verify understanding before proceeding

### 2. **Documentation Standardization**
- ✅ Follow established templates
- ✅ Consistent formatting across all docs
- ✅ Version control and change tracking
- ✅ Clear structure and organization

### 3. **Planning Precision**
- ✅ Define clear requirements first
- ✅ Break down into specific, actionable steps
- ✅ No vague or ambiguous tasks
- ✅ Verify scope before execution

### 4. **Persistent Consistency**
- ✅ Maintain same standards across sessions
- ✅ Reference previous decisions
- ✅ Follow established patterns
- ✅ No drift from standards

---

## Implementation Strategy

### Documentation Standards
1. **Template-Based Approach**
   - Use standard templates for all documentation
   - Enforce consistent structure
   - Validate against standards before saving

2. **Quality Gates**
   - Check completeness before finalizing
   - Verify formatting compliance
   - Ensure no placeholders or TODOs

3. **Version Control**
   - Track all changes
   - Maintain change history
   - Document rationale for changes

### Planning Precision
1. **Requirement Gathering**
   - Ask clarifying questions first
   - Document all requirements explicitly
   - Verify understanding before planning

2. **Task Breakdown**
   - Specific, measurable tasks
   - Clear acceptance criteria
   - No ambiguous descriptions

3. **Validation**
   - Review plan with user before execution
   - Confirm scope and approach
   - Adjust based on feedback

---

## Workflow Enforcement

### Before Starting Any Task:
1. ✅ Understand requirements (ask if unclear)
2. ✅ Check existing standards/templates
3. ✅ Plan approach (get approval if needed)
4. ✅ Execute with precision
5. ✅ Verify against standards

### Documentation Creation:
1. ✅ Use appropriate template
2. ✅ Follow formatting standards
3. ✅ Include all required sections
4. ✅ Validate completeness
5. ✅ Save with proper naming

### Planning Documents:
1. ✅ Gather all requirements first
2. ✅ Break into specific tasks
3. ✅ Define acceptance criteria
4. ✅ Get confirmation before proceeding
5. ✅ Track progress precisely
